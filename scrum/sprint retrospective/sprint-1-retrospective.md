# Sprint Retrospective – Sprint 1

**Projeto:** Avalia+
**Período da Sprint:** Março de 2026

---

# Objetivo da Retrospectiva

A Sprint Retrospective tem como objetivo analisar o processo de trabalho da equipe durante a sprint, identificando pontos positivos, dificuldades e possíveis melhorias para as próximas etapas do projeto.

---

# O que funcionou bem

* Organização inicial do projeto no GitHub
* Boa comunicação entre os integrantes da equipe
* Definição das funcionalidades iniciais do sistema
* Estruturação inicial da documentação do projeto

---

# Dificuldades encontradas

* Dificuldade inicial com algumas tecnologias que serão utilizadas no projeto
* Limitações de tempo para estudo e desenvolvimento devido às atividades acadêmicas
* Ajustes necessários na divisão de tarefas da equipe

---

# Melhorias para a próxima Sprint

Para melhorar o andamento do projeto nas próximas sprints, a equipe definiu as seguintes ações:

* Melhor organização das tarefas da sprint
* Continuidade nos estudos das tecnologias utilizadas no sistema
* Avançar na modelagem do sistema antes do início do desenvolvimento
* Manter registros atualizados das atividades realizadas

---

# Conclusão

A Sprint 1 foi fundamental para estabelecer as bases do projeto Avalia+. Apesar das dificuldades iniciais, a equipe conseguiu estruturar o projeto e preparar o terreno para o avanço do desenvolvimento nas próximas sprints.
