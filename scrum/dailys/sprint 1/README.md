# Dailys – Sprint 1

Esta pasta contém os registros das **Daily Scrums realizadas durante a Sprint 1** do projeto **Avalia+**.

As Daily Scrums são reuniões rápidas realizadas pela equipe para acompanhar o andamento do projeto, identificar impedimentos e alinhar as próximas atividades.

Durante cada daily, os integrantes responderam principalmente três pontos:

- O que foi realizado desde a última atualização
- O que será feito até a próxima atualização
- Se existe algum impedimento que possa impactar o andamento do projeto

---

## Período da Sprint

Sprint 1 corresponde ao período inicial de organização do projeto, incluindo:

- Configuração do ambiente de desenvolvimento
- Criação do repositório do projeto
- Levantamento inicial de requisitos
- Definição dos tipos de usuários
- Elaboração da introdução do projeto
- Criação do Product Backlog

---

## Objetivo da Sprint 1

Estabelecer a **base inicial do projeto Avalia+**, organizando o repositório, estruturando a documentação inicial e definindo as funcionalidades que farão parte do sistema.
