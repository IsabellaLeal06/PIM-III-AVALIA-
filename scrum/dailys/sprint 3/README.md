# Sprint 3 – Projeto Avalia+

Esta pasta contém os registros relacionados à **Sprint 3** do projeto Avalia+.

A Sprint 3 marca o início do **desenvolvimento do sistema**, com foco na implementação das funcionalidades principais, integração entre as partes e evolução técnica da aplicação.

---

## Conteúdo da pasta

- dailys/
- sprint-3-planning.md
- sprint-3-review.md
- sprint-3-retrospective.md
- sprint-3-backlog.md

---

## Objetivo da Sprint

Implementar as primeiras funcionalidades do sistema Avalia+, iniciando o desenvolvimento do **backend**, **frontend** e **banco de dados**, além de promover a integração entre essas camadas.

---

## Principais atividades

- Desenvolvimento do sistema de autenticação (login)
- Implementação do cadastro de usuários
- Estruturação inicial do banco de dados
- Desenvolvimento das primeiras telas da aplicação
- Integração entre frontend, backend e banco de dados

---

## Organização da equipe

Durante esta sprint, as atividades foram divididas conforme as áreas do projeto:

- **Backend:** responsável pela lógica do sistema e regras de negócio  
- **Frontend:** responsável pela interface e experiência do usuário  
- **Banco de Dados:** responsável pela estrutura e persistência dos dados  
- **Documentação:** acompanhamento e atualização do projeto  

---

## Resultados esperados

Ao final da Sprint 3 espera-se:

- Sistema de login funcional  
- Estrutura inicial do banco de dados implementada  
- Interface inicial da aplicação desenvolvida  
- Integração básica entre as partes do sistema  

---

## Importância da Sprint

A Sprint 3 é fundamental pois representa a transição da fase de **planejamento e modelagem** para a fase de **desenvolvimento prático**, permitindo validar as decisões tomadas nas sprints anteriores.
