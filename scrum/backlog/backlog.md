# Backlog – Projeto Avalia+

## Descrição

Este documento apresenta o **Backlog do projeto Avalia+**, contendo a lista organizada e priorizada de todas as atividades necessárias para o desenvolvimento do sistema.

O backlog inclui funcionalidades do sistema, atividades de modelagem e itens de documentação, servindo como base para o planejamento e execução das Sprints.

---

## Funcionalidades do Sistema

| ID   | Funcionalidade            | Descrição                                                           | Prioridade |
| ---- | ------------------------- | ------------------------------------------------------------------- | ---------- |
| PB01 | Autenticação de usuários  | Permitir login no sistema utilizando e-mail e senha                 | Alta       |
| PB02 | Cadastro de usuários      | Cadastro e gerenciamento de usuários (realizado pelo administrador) | Alta       |
| PB03 | Gerenciamento de questões | Cadastro, edição e exclusão de questões                             | Alta       |
| PB04 | Geração de simulados      | Criação automática de simulados com questões aleatórias             | Alta       |
| PB05 | Realização de simulados   | Permitir que alunos respondam simulados                             | Alta       |
| PB06 | Correção automática       | Correção automática após finalização do simulado                    | Alta       |
| PB07 | Histórico de simulados    | Armazenamento dos simulados realizados                              | Média      |
| PB08 | Relatórios de desempenho  | Geração de relatórios individuais e por turma                       | Média      |
| PB09 | Dashboards de desempenho  | Exibição de indicadores visuais                                     | Média      |
| PB10 | Controle de permissões    | Restrição de acesso conforme perfil do usuário                      | Alta       |
